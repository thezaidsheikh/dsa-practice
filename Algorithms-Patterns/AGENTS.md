# AGENTS.md - Algorithms-Patterns

This folder contains core algorithm pattern explanations and conceptual learning material.

## Folder Structure

- `graph/` - Graph-related concepts (traversal, topological sort, etc.)
- Other pattern files at root level

## Content Style

Each pattern file should include:

1. **Definition** - Clear one-liner of what the pattern is
2. **When To Use** - Problem recognition cues and keywords
3. **Why It Works** - Intuition and reasoning
4. **Algorithm Steps** - Numbered steps for the approach
5. **Visual Example** - Diagram or ascii illustration
6. **Code Implementation** - Java code block
7. **Complexity Analysis** - Time and space
8. **Comparison** - vs related patterns when relevant
9. **Common Problems** - LeetCode problem references

## Naming Conventions

- Use kebab-case: `graph.md`, `kahns-algorithm.md`
- Use descriptive names that match the pattern name
- Match existing style in sibling files

## Code Standards

- Use Java for implementations
- Use `class Solution` for LeetCode-style code
- Include meaningful variable names
- Format inside fenced code blocks

## What To Avoid

- Do not add single problem notes here
- Do not use overly casual language
- Do not skip complexity analysis
- Do not omit the "why it works" explanation
